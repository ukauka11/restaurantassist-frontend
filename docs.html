Endpoints

Core API routes
  • GET /health — quick uptime check
  • GET /me — your business + plan + subscription status
  • POST /analyze — send a review, get AI response + tags
  • GET /summary — daily/all-time summary of your saved reviews
If subscription is not active, protected routes return 402.

Headers

Required for protected routes
x-api-key: cust_...
Your API key identifies your business. Keep it private.

POST /analyze body

JSON payload
{
  "review_text": "Food was great but the wait was long.",
  "platform": "google",
  "customer_name": "Optional",
  "order_number": "Optional",
  "run_id": "Optional"
}
platform can be google, yelp, facebook, or other.

What you get back

Response contains a ready-to-post reply
{
  "business_id": "your-business",
  "record": {
    "sentiment": "negative",
    "urgency": "medium",
    "category": "speed_wait_time",
    "tags": ["wait_time"],
    "reply": "Thanks for your feedback! ...",
    "next_steps": ["Review staffing at peak hours", "..."]
  }
}

Troubleshooting

Common issues
  • 401 Unauthorized — your x-api-key is missing/incorrect
  • 402 Subscription inactive — payment failed or subscription canceled
  • Network error — check API Base URL and internet
  • Email went to Spam — click “Not spam” to train your inbox
